# SGLang backend

## Scope and support status

The Stage A backend is a persistent SGLang `0.5.10.post1` HTTP measurement
wrapper for transformation-free `full_kv`. It supports the audited Qwen
checkpoints below. A source registration is not a runtime claim; each row is
only GPU-supported after its exact smoke command succeeds.

| Model | Immutable revision | Source support | GPU status |
|---|---|---|---|
| `Qwen/Qwen2.5-VL-3B-Instruct` | `66285546d2b821cf421d4f5eb2576359d3770cd3` | `srt/models/qwen2_5_vl.py:554` | Stage A FullKV verified by Slurm job `17160103` |
| `Qwen/Qwen2.5-VL-7B-Instruct` | `cc594898137f460bfe9f0759e9844b3ce807cfb5` | same model class | Stage A FullKV verified by Slurm job `17160299` |

Other models fail before server launch. Stage B native MosaicKV is
unsupported for the audited version; see [sglang_native_blocker.md](sglang_native_blocker.md).

## Correctness-first server profile

The exact command is generated by
`mosaickv.backends.sglang_runtime.build_server_command` and stored verbatim in
every trace and run manifest. SGLang's TVM-FFI JIT kernels require a C++20
standard library; the Slurm entrypoints explicitly load `gcc/11.4.0` because
the node-default GCC 8.5 lacks `<version>`. The compiler version and `CC`/`CXX`
environment are stored in each trace. `TVM_FFI_CACHE_DIR` is set below the
external MosaicKV cache root, never the home directory. The initial profile
sets:

- `--enable-deterministic-inference` and `--attention-backend triton`;
- `--disable-overlap-schedule`;
- `--disable-cuda-graph`;
- `--skip-server-warmup`;
- `--stream-interval 1`;
- `--enable-metrics` and `--enable-cache-report`;
- `--page-size 1`, which removes page-padding ambiguity from FullKV active-byte
  accounting; and
- `--disable-fast-image-processor`, matching the slow checkpoint processor
  used by the HF reference instead of accepting the Transformers 5
  fast-by-default behavior; and
- `--model-impl sglang`, preventing an unrecorded Transformers fallback.

No optimized production profile is enabled. Such a profile must be a separate
configuration and may only be compared after token parity is established.

## Validation record and parity boundary

The pinned 175-distribution environment passed its import, lock, and CUDA
matrix checks on an A100-SXM4-80GB with driver 595.71.05. The final 3B job
`17160103` completed two distinct requests, deterministic repeated trials,
the post-intervening A-B-A isolation probe, Radix observations, GPU-memory
sampling, and exact active-byte checks. The corresponding HF eager job
`17160104` used the same immutable model/input hashes, slow processor, BF16,
fixed 16-token output, and explicit greedy settings.

Final 7B job `17160299` used the same correctness profile and completed the
two-input deterministic, telemetry, accounting, and A-B-A isolation gates from
the exact local revision. No 7B HF parity claim is made.

HF/SGLang output parity was **not established**: both controlled sample
comparisons retain a failed status in
[`results/sglang/`](../results/sglang/). This is consistent with the audited
SGLang-native Triton model path differing numerically from HF eager execution;
it is not rewritten as a pass. Consequently no optimized SGLang profile has
been added, and HF/SGLang latency rows must not be combined. Stage A support
means measured SGLang FullKV execution, not cross-backend token equivalence.

These runs came from a dirty development worktree and are validation evidence,
not paper-eligible experimental results.

## Shared prompts and deterministic generation

The wrapper calls the same `AutoProcessor.apply_chat_template` boundary used
by the HF and vLLM harnesses. Ordered images/videos and prompt text are not
rewritten. Media is only serialized to a JSON-safe base64 value for transport.
The rendered prompt, raw media, and locally tokenized rendered prompt IDs are
stored in the trace. Requests use temperature 0, top-p 1, the recorded seed,
repetition penalty 1, zero presence/frequency penalties, and `ignore_eos=true`
to match the HF harness's fixed-length explicit greedy loop. The complete
effective sampling object is stored in every trace.

Each sample is sent at least twice through the same server. Generated token
IDs must be identical or the sample becomes a failure row. Every HTTP call has
a unique request ID and omits `session_params` and `conversation_id`. For a
multi-input run, the first input is run once more after the other inputs; its
token IDs must match the original output. This A-B-A isolation probe detects
intervening request state affecting the anchor while still allowing SGLang's
intended immutable Radix prefix reuse. It tests state isolation, not whether a
model's truncated answer happens to contain the reference string.

## Measurements

Each raw trial records:

- client-observed TTFT from the first streamed output token;
- every token timestamp and inter-token interval;
- decode duration, request duration, and token throughput;
- SGLang `meta_info` prompt, completion, cached-token, queue, prefill, and
  server end-to-end observations where emitted;
- `sglang:cache_hit_rate`, `sglang:cached_tokens_total`,
  `sglang:gen_throughput`, and `sglang:token_usage` from `/metrics`;
- process-tree GPU memory from `nvidia-smi` before and during the request; and
- the server's reserved weight/KV/graph memory from `/server_info`.

SGLang exposes no per-request multimodal encoder-cache hit counter in the
audited source. Repeated-request timing is recorded, but it is not labeled a
cache hit.

The logical active cache after `N` output tokens contains all prompt positions
and the first `N-1` decoded positions: the last sampled token has not yet been
fed back through the model. Therefore:

```text
active_positions = prompt_tokens + max(completion_tokens - 1, 0)
bytes_per_position = 2 * layers * KV_heads * head_dim * dtype_bytes
active_KV_bytes = active_positions * bytes_per_position
```

The exact checkpoint geometry and every operand are stored in the trace.
Reserved pool memory is reported separately and is never substituted for
active bytes.

## Commands

Environment verification on an allocated GPU:

```bash
/scratch/djy8hg/env/mosaickv/bin/python scripts/verify_envs.py \
  --environment sglang \
  --lock env/sglang/requirements.lock \
  --require-cuda
```

Submit the offline 3B correctness smoke:

```bash
sbatch slurm/sglang_fullkv.sbatch
```

Run the 7B checkpoint after its immutable snapshot is available:

```bash
MOSAICKV_SGLANG_CONFIG=configs/sglang_fullkv_7b.yaml \
  sbatch slurm/sglang_fullkv.sbatch
```

An explicit first-time environment setup plus smoke is available as:

```bash
sbatch slurm/sglang_setup_and_fullkv.sbatch
```

Model caches live below `/scratch/djy8hg/cache/mosaickv`; `HF_TOKEN`, when
needed, is read only from the process environment. The setup job is the only
Slurm entrypoint that mutates the environment.

## Validation gates

`scripts/verify_sglang_backend.py` rejects traces that lack correctness flags,
exact byte accounting, deterministic repeated tokens, cache measurements, GPU
memory, globally unique request IDs, or the post-intervening A-B-A isolation
probe. HF parity is compared from
immutable traces with `scripts/compare_sglang_hf_fullkv.py`. A failed parity
comparison remains a failed artifact; it must not be rewritten as a passing
result.
